# Chestnut Editor v0.4.0

---

## English

A desktop editor for local Markdown and Excalidraw files — remote image support in bundles, sharper image editing, and a cleaner file tree.

Chestnut Editor reads and writes plain files in a folder you choose. No proprietary database, no cloud required.

### Download

| Item | Details |
|------|---------|
| **Installer** | `Chestnut_0.4.0_x64-setup.exe` (Windows x64, NSIS) |
| **Requirements** | 64-bit Windows 10+; WebView2 (downloaded during install if missing) |
| **Default vault** | `~/.chestnut` on first launch |

**Build output:**

`apps/desktop/src-tauri/target/x86_64-pc-windows-msvc/release/bundle/nsis/Chestnut_0.4.0_x64-setup.exe`

### What's New in v0.4.0

#### Remote & cloud images

- **Export / import** now downloads `https://` image URLs and cloud attachment links before packaging or converting to vault notes
- **Live editor paste**: a single-line Markdown image (`![alt](https://… "caption")`) is parsed and rendered as an image block instead of plain text
- Cloud attachment URLs are resolved back to vault paths when possible

#### Image editing

- **First-time captions** appear under the image immediately (no delayed or misplaced caption)
- Captions are read from the editor model, not DOM `alt` text
- **Ctrl+C** on a selected image copies Markdown (`![caption](path)`) to the clipboard

#### File tree & sidebar

- Toggle **Show / hide `_pic` image folders** from the sidebar (hidden by default for a cleaner tree)
- Slimmer sidebar scrollbar styling
- **Rename** behavior is more reliable when renaming folders and files
- Sibling names with numeric suffixes sort naturally (`Note 2` before `Note 10`)
- Improved **code block** styling in the live editor

#### Import UI

- Toolbar **import icon** uses an inbox-with-arrow design (reads as import, not download)
- Shorter import tooltip; removed the “md and images in same directory” hint

### Upgrading from v0.3.0

- No vault migration required; existing notes and `_pic` folders are unchanged
- Markdown export/import now fetches remote images at export/import time (network required for notes that reference URLs)
- `_pic` folders are hidden in the file tree by default; use the sidebar toggle to show them

### Highlights from v0.3.0 (still included)

- Markdown bundle export/import; editor shortcuts, quick open, plain-text copy
- File tree folder selection, create-in-folder, instant rename on create
- Excalidraw locale sync; PDF export; plugins (experimental)

## 简体中文

面向本地 Markdown 与 Excalidraw 的桌面编辑器 —— 远程图片导入导出、图片编辑更顺手，文件树更清爽。

Chestnut Editor 在你选择的文件夹中直接读写普通文件，无需专有数据库，也不依赖云端。

| **默认知识库** | 首次启动打开 `~/.chestnut` |

**构建产物：**

`apps/desktop/src-tauri/target/x86_64-pc-windows-msvc/release/bundle/nsis/Chestnut_0.4.0_x64-setup.exe`

### v0.4.0 新内容

#### 远程与云端图片

- **导出 / 导入** 时会先下载 `https://` 图片链接及云端附件 URL，再打包或转为知识库笔记
- **实时模式粘贴**：单行 Markdown 图片语法（`![描述](https://… "说明")`）会解析为图片块，而非纯文本
- 云端附件 URL 在可能时解析回 vault 内路径

#### 图片编辑

- **首次添加图片描述** 会立即显示在图片下方（修复延迟与位置错位）
- 描述从编辑器模型读取，不再误用 DOM `alt` 文本
- 选中图片后 **Ctrl+C** 可复制为 Markdown（`![描述](路径)`）

#### 文件树与侧栏

- 侧栏可切换 **显示 / 隐藏 `_pic` 图片文件夹**（默认隐藏，树更简洁）
- 侧栏滚动条样式更细
- **重命名** 文件夹与文件时行为更稳定
- 带数字后缀的同级名称按数值排序（如 `笔记 2` 排在 `笔记 10` 前）
- 实时编辑器 **代码块** 样式优化

#### 导入界面

- 顶栏 **导入图标** 改为收件箱 + 向下箭头（更易识别为导入而非下载）
- 精简导入 tooltip，去掉「md 与图片同目录」说明

### 从 v0.3.0 升级说明

- 无需迁移知识库；现有笔记与 `_pic` 目录保持不变
- 含远程 URL 的笔记在导出/导入时会联网下载图片
- 文件树默认隐藏 `_pic` 文件夹；可在侧栏开关中显示

### v0.3.0 已有能力（本版保留）

- Markdown 包导入导出；编辑器快捷键、快速打开、纯文本复制
- 文件树文件夹选中、在选中目录创建、新建后立即重命名
- Excalidraw 语言跟随应用；PDF 导出；插件（实验性）
